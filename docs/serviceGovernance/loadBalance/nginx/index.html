    Nginx反向代理 #

    类型 #

    • 带权重的round-robin算法是基础
    • hash负载均衡算法 ip-hash算法 -> real-ip hash算法 -> 自定义可以hash的参数（比如?userName）

      问题: 如果有upstream的机器宕机， hash算法还会路由到这台机器 解决方案：使用一致性hash(consistent),hash 环

    • least-connection算法， 如果所有节点的connection都一致， 会退化成为round-robin算法。

    可扩展立方体 #

    1. X-axis 基于round-robin或者least-connected算法分发请求 -> 相对简单
    2. Y-axis 基于URL对功能进行分发。 -> 相对复杂
    3. Z-axis 将用户IP地址或者其他信息映射到某个特定的服务或者集群 -> 相对简单

    多种协议反向代理 #

    1. tcp udp 透传
    2. http -> memcached , scgi, fastcgi, uwsgi, grpc, http, websocket
